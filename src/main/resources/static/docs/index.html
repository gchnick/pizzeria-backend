Descripción

Esta es una interfaz de programación de aplicaciones (API o API web) que se ajusta a los límites de la arquitectura REST (API de RESTful). Esta diseñada para entender la solicitud de la Pizzería Don Remolo, y asi proveer un sistema de solicitud de pedidos y envíos a la carta. A la vez que sus administradores puedan mantener al dia los productos ofrecidos.

Este documento describe como se puede usar este backend API REST. El equipo de frontend puede usar esta información para poder comunicarse con una base de datos.

1. General

1.1. Verbos HTTP

HTTP define un conjunto de métodos de petición para indicar la acción que se desea realizar para un recurso determinado. Se describirán a continuación los verbos HTTP implementados en esta API y como se usan.

Table 1. Verbos HTTP usados en esta API
Verbo Uso

GET

Se utiliza para recuperar un recurso (idempotente)

POST

Se utiliza para crear un nuevo recurso

PUT

Se utilizada para actualizar un recurso existente. Este reemplaza todas las representaciones actuales del recurso existente con la carga útil de la petición (idempotente)

PATCH

Se utilizada para aplicar modificaciones parciales a un recurso existente.

DELETE

Se utilizada para eliminar un recurso existente (idempotente)

2. Administrador

El usuario con permisos de administrador mantener el actualizada la información de los productos ofrecidos en la pizzería. A demás podrá ver los pedidos que no ha sido despachados.

Usted puede controlar a todos los usuarios registrados como administradores. Podrá ver la lista actual de administradores. Agregar o eliminar un administrador.

2.1. Añadir un nuevo administrador

POST /api/v1/users

Añadir un nuevo usuario administrador del sistema.

2.1.1. Autorización

Se requiere token de acceso de usuario.

2.1.2. Campos de solicitud (request fields)

Path Type Optional Description

id

Integer

true

Identificación única.

fullName

String

false

Nombre completo del usuario.

Size must be between 3 and 100 inclusive.

email

String

false

Correo electrónico del usuario. Este se usara como username para poder solicitar un token de acceso.

Must be a well-formed email address.
Size must be between 3 and 100 inclusive.

password

String

true

La contraseña debe tener al menos: un dígito, una letra en mayúscula y minúscula, un signo especial [@@$%^&+=]. Tener 8 caracteres de tamaño y no tener ningún espacios.

role

String

false

El valor siempre debe ser ROLE_ADMIN.

Must be one of [ROLE_ADMIN].

2.1.3. Campos de respuesta (response fields)

Path Type Description

data.user

Object

Información de usuario creado
